The Event table represents events which we have reason to believe actually happened. The very similar Eventum table is used to represent a record of those events, as recorded in the Reference statements. Individuals are linked to Events using the IndividualEvent table, which will also indicate the role played by the Individual. In a similar fashion, the EventumPersona table can link a Persona to an Eventum and record their role. Both the IndividualEvent and EventumPersona tables share the same EventTypeRole table. The EventType table is also shared by both the Event and Eventum tables. Event's are linked to their Eventum records by the EventEventum table. The FamilyEvent table holds a separate link between an Event and the Family for family event types. Note that, as Individual and Persona records are linked by the IndividualPersona table, this can set up a redundant circlar link - this is explored further in the Event - Eventum overview page.

4Other FamilyAny family event in which both partners are equally equally prime, other than a official union event. It may occur before (engagment) or after (divorce) the union event.
8PersonalThese are events which last the lifetime of an individual. Although the details may change over time, at any given time they normally have a single state. ie Occupation, Physical appearance or Residency
7OtherAnything which does not fit into the other Groups

A list of Event Types and the Roles normally associated with them is maintained as part of the core data - see the Core Data: EventTypes page for a current list. The user is free to extend this list as required. All Types are categorised into one of the groups shown opposite. This list cannot be extended by the user, although a future database version may.

The Group designated for an EventType has a significant effect on the way the Event is handled by the application.
