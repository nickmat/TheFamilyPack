Subsystems

We can break the database design into four parts, Administration, Evidence, Interpretation and Conclusions.

Administration

The Administration area of the database is the least developed section, only the basic minimum is being implemented. A version table will enable an automatic update process to be developed, for when changes are made to the database design. The Contact section helps to form a bridge between the Administration area with the other areas of the database. The User section is the area where the database can hold initial start up setting.

Evidence

The evidence section centres around the Reference table which is suported by the Source, Citation and Repository subsystems.

Interpretation

This is the area where the main work is done. The information presented in the Evidence section is broken down into its constituent parts: Sources, Dates, Places, Names, Relationships, Attributes and Events. Events and Personas are used to glue all these entities back together again so the computer program can make use of it.

Conclusions

The Individuals and Family relationships are the results of the interpretion of the evidence, using the LinkPersona and LinkEvent tables tie things back to the interpretation.
